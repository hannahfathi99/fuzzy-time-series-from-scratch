# Contributing

Thank you for your interest in contributing to this repository.

This project presents a complete from-scratch implementation of **First-Order Fuzzy Time Series (FOFTS)** and **Higher-Order Fuzzy Time Series (HOFTS)** forecasting models. Contributions that improve readability, reproducibility, documentation, or implementation quality are welcome.

## Ways to Contribute

You can contribute by

* fixing bugs
* improving documentation
* optimizing implementation
* adding unit tests
* improving visualization
* extending forecasting methods
* improving code readability
* reporting issues

## Development Setup

Clone the repository

```bash
git clone https://github.com/hannahfathi99/fuzzy-time-series-from-scratch.git
cd fuzzy-time-series-from-scratch
```

Create a virtual environment

```bash
python -m venv .venv
```

Activate it

Linux/macOS

```bash
source .venv/bin/activate
```

Windows

```bash
.venv\Scripts\activate
```

Install dependencies

```bash
pip install -r requirements.txt
```

Launch the notebook

```bash
jupyter notebook notebooks/Fuzzy_Project1.ipynb
```

## Coding Guidelines

Please follow these principles.

* Use descriptive variable names.
* Write clear docstrings.
* Preserve reproducibility.
* Maintain compatibility with Python 3.10 or newer.
* Avoid introducing external fuzzy time series libraries.
* Follow PEP 8 style conventions whenever possible.

## Pull Requests

Before opening a pull request, please ensure that

* the notebook executes without errors;
* generated results remain reproducible;
* documentation is updated if necessary;
* new features include appropriate explanations.

## Reporting Issues

When reporting a bug, please include

* operating system
* Python version
* package versions
* steps to reproduce
* expected behavior
* observed behavior

Thank you for helping improve this project.
